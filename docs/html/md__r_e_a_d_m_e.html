Antenna Mapping Project

Overview

This project is a practical work for the Advanced Data Structures course (ESI). It involves reading a matrix from a file where dots represent empty spaces, and characters represent antennas. The program calculates the effect positions of antennas using specific formulas and provides functionalities to insert, remove, print antennas, and clear the lists.

Features

  • Load antennas from an input file.
  • Print the current antenna map.
  • Print the effect positions of antennas.
  • Insert a new antenna.
  • Remove an existing antenna.
  • Save antennas to an output file.
  • Save effect positions to an output file.

Effect Position Calculation

Effect positions are calculated using the formulas:

  • L1 = (2 * row1 - row2, 2 * col1 - col2)
  • L2 = (2 * row2 - row1, 2 * col2 - col1)

Project Structure

  • src/: Contains source code files (main.c, menu.c, io_ops.c, antenna.c, antenna_effect.c).
  • include/: Contains header files (menu.h, io_ops.h, antenna.h, antenna_effect.h).
  • docs/: Contains documentation files generated by Doxygen (configfile.txt, html/index.html).
  • input/: Contains input files with initial matrix configurations.
  • CMakeLists.txt: CMake configuration file.

Source Files

  • main.c: Entry point of the program; initializes and displays the menu, and handles user interactions.
  • menu.c: Implements menu-related functionalities.
  • io_ops.c: Handles input/output operations, including reading and writing files.
  • antenna.c: Manages antenna operations (loading, inserting, removing, printing).
  • antenna_effect.c: Calculates and manages antenna effect positions.

Header Files

  • menu.h: Declares functions for menu handling.
  • io_ops.h: Declares I/O operation functions.
  • antenna.h: Declares antenna management functions.
  • antenna_effect.h: Declares functions for effect position calculations.

Documentation

Generated using Doxygen:

  • configfile.txt: Doxygen configuration file.
  • html/index.html: Main entry for HTML documentation.

Input File Format

Each input file must adhere to the following format:

  • Matrix representation using dots (.) for empty spaces and other characters for antennas.
  • Each row is on a separate line.
  • The matrix must be rectangular.

Example:

. . A . .
. B . . .
. . . C .
. . . . D

Requirements

  • CMake Minimum Version: 3.30

Building the Project

  1. Clone the repository.
  2. Open the project in CLion.
  3. Configure CMake settings.
  4. Build the project.
